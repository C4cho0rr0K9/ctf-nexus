// CTF NEXUS — GUIA DE RESOLUCIÓN

Walkthrough: NEXUS v3.7.1

URL del reto: http://localhost:9090/

Dificultad: MEDIA

IMPORTANTE: Los valores de session_id y auth_token cambian cada vez. Usa los que te salgan a ti, no copies los del ejemplo.

>> Si accedes desde otra máquina, reemplaza localhost por la IP del servidor.

COMANDO ÚNICO — Resuelve todo automaticamente: 

$ SID=$(curl -s -X POST -H "X-Bootstrap: enable" http://localhost:9090/init | python3 -c "import sys,json;print(json.load(sys.stdin)['session_id'])") && TOK=$(curl -s -X POST -H "Content-Type: application/json" -d "{\"session_id\":\"$SID\"}" http://localhost:9090/portal | python3 -c "import sys,json;print(json.load(sys.stdin)['auth_token'])") && curl -s -H "Authorization: Bearer $TOK" http://localhost:9090/flag
PASO 1 // RECONOCIMIENTO INICIAL

Accede a la página principal:

$ curl -s http://localhost:9090/

Busca en la salida estas líneas clave:

[2026-07-04 14:00:05] NEXUS::warning Endpoints sensibles detectados: /init, /portal, /flag
[2026-07-04 14:00:06] NEXUS::hint Revisa el mapa del sitio antes de continuar
[2026-07-04 14:00:08] NEXUS::debug WC1Cb290c3RyYXA6IGVuYWJsZQ==

El string WC1Cb290c3RyYXA6IGVuYWJsZQ== es base64. También se mencionan 3 endpoints: /init, /portal y /flag.

PASO 2 // DECODIFICAR BASE64

Decodifica el mensaje oculto:

$ echo "WC1Cb290c3RyYXA6IGVuYWJsZQ==" | base64 -d

Resultado:

X-Bootstrap: enable

Es un header HTTP. El nombre del header es X-Bootstrap y el valor es enable. Se usa en /init.

PASO 3 // ROBOTS.TXT

Consulta el mapa del sitio:

$ curl -s http://localhost:9090/robots.txt

Resultado:

User-agent: *
Disallow: /init
Disallow: /portal
Disallow: /flag

Confirma los 3 endpoints. Orden: /init → /portal → /flag.

PASO 4 // AUTENTICACIÓN EN /init

GET da error — requiere POST con header:

$ curl -s -X POST -H "X-Bootstrap: enable" http://localhost:9090/init

Respuesta exitosa:

{"status":"SESSION_CREATED","session_id":"dbbdddc99d5637e17d365e87ea223459","next":"/portal"}

[*] Recibes un session_id. Copia ese valor para el siguiente paso.

PASO 5 // INTERCAMBIAR SESSION POR TOKEN

Usa el session_id en /portal:

$ curl -s -X POST -H "Content-Type: application/json" \
  -d '{"session_id":"dbbdddc99d5637e17d365e87ea223459"}' \
  http://localhost:9090/portal

Respuesta exitosa:

{"status":"AUTH_GRANTED","auth_token":"4ddd3cacd8a21f6999f30ef7c06f25fc2bad4b6553424358535425f35eee5d0a","next":"/flag"}

[*] Recibes un auth_token. Copia este valor — es la clave final.

PASO 6 // OBTENER LA FLAG

Usa el token como Bearer en /flag:

$ curl -s -H "Authorization: Bearer 4ddd3cacd8a21f6999f30ef7c06f25fc2bad4b6553424358535425f35eee5d0a" \
  http://localhost:9090/flag

Respuesta exitosa:

{"status":"ACCESS_GRANTED","flag":"CTF{...}"}

[*] La flag aparece en el campo flag.

PASO 7 // VERIFICAR LA FLAG

Verifica via endpoint /submit:

$ curl -s -X POST -H "Content-Type: application/json" \
  -d '{"flag":"CTF{...}"}' \
  http://localhost:9090/submit

Respuesta correcta:

{"status":"correct","message":"ACCESS GRANTED // Flag verificada correctamente. Has completado el reto NEXUS."}

[*] Reto completado.
